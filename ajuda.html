❓ Ajuda

Guia completo do WebTikTokLiveSNES — do primeiro acesso ao mapeamento avançado de gifts.

🚀 Primeiros passos

O WebTikTokLiveSNES roda diretamente no browser — sem instalação, sem extensão. Você só precisa de uma ROM SNES e de estar ao vivo no TikTok.

⚠️
ROMs não estão incluídas O emulador usa arquivos .sfc / .smc que você deve possuir legalmente. Use ROMs homebrew gratuitas ou arquivos de cartuchos que você já comprou.
1
Abra o app Acesse app.html no Chrome ou Edge. O Firefox também funciona, mas o Chrome tem melhor suporte a TTS em pt-BR.
2
Carregue a ROM Arraste um arquivo .sfc / .smc para a área pontilhada, ou clique nela para abrir o seletor de arquivo. O emulador inicia automaticamente.
3
Conecte ao TikTok Live Cole a URL do Zerody OBS na sidebar (ou preencha @username + token) e clique Conectar. Você precisa estar ao vivo para receber eventos.
4
Escolha o preset do jogo Selecione o preset correto na sidebar (ex: mario, dk). Os gifts mapeados aparecem nos botões de teste logo abaixo.
5
Teste sem estar ao vivo Use os botões 🧪 Testar Gifts para simular eventos e verificar se as teclas chegam ao emulador antes de ir ao ar.

🔴 Conectar ao TikTok Live

A conexão usa o serviço Zerody como bridge para o chat ao vivo do TikTok (API não-oficial). Há dois modos:

Modo URL completa (recomendado)

Acesse tiktok-chat-reader.zerody.one, faça o login e copie a URL da seção OBS Browser Source. Cole essa URL no campo da sidebar e clique Conectar.

💡
Por que usar a URL completa? O Zerody exige um captcha na primeira autenticação. Ao colar a URL já autenticada você pula esse passo e conecta diretamente.

Modo @username + token

Preencha os campos @username e Token manualmente com os dados obtidos no painel do Zerody e clique Conectar.

ℹ️
Você precisa estar ao vivo O app conecta ao chat em tempo real. Se você não estiver transmitindo, a conexão será estabelecida, mas nenhum evento chegará.

🎁 Gifts → Ações

Cada gift enviado no TikTok dispara uma ação no emulador. O mapeamento fica em ✏️ Editar → aba Gifts. O nome do gift é normalizado automaticamente (minúsculas, sem acentos, sem símbolos) antes de buscar no mapa.

Exemplo: gift "Rosa" → agachar no Mario

CampoValorDescrição
Nome do giftrosaNormalizado: minúsculas, sem acentos
SequênciaDOWN:600Segura ↓ por 600 ms
Label🌹 Agachou!Texto exibido no banner
TTSRosa! Mario agachou.Narração falada em pt-BR

Normalização de nomes

O nome do gift passa por normalizeGift() antes da busca:

"Interstellar ✨" → "interstellar"
"Coração 💜"    → "coracao"
"Rose"           → "rose"

Use o nome normalizado como chave no mapeamento.

⌨️ Sequências de teclas

Uma sequência define quais botões do SNES são pressionados, por quanto tempo, e a pausa entre eles.

Formato

TECLA:holdMs:pausaMs   TECLA2:holdMs   ...

holdMs — tempo em ms que a tecla fica pressionada (padrão: 100)
pausaMs — pausa após soltar antes da próxima tecla (padrão: 80)

Teclas disponíveis

AliasBotão SNES
BBotão B
YBotão Y
ABotão A
XBotão X
LGatilho L
RGatilho R
STARTStart
SELECTSelect
UP DOWN LEFT RIGHTD-pad

Exemplos

# Pular (B por 150ms)
B:150

# Agachar (↓ por 600ms)
DOWN:600

# Hadouken SF2: ↓ + ↘ + → + Soco forte
DOWN:80:50 DOWN:80:50 RIGHT:80:50 Y:150

# Combo com pausa longa entre teclas
B:200:400 A:200:400 B:200

✨ Ações especiais

Além de sequências de teclas, existem ações especiais configuradas pelo campo special:

specialCampo extraEfeito
rewind rewindMs (ex: 2500) Mantém a tecla de rewind pressionada por N ms — volta o jogo no tempo
slowmo slowmoMs (ex: 5000) Ativa o slow motion nativo do RetroArch por N ms
reset — Reinicia o jogo (equivale a resetar o cartucho)
powerup powerupIdx (0, 1, 2…) Aplica o power-up de índice N, escrevendo direto na WRAM
💡
Rewind buffer O rewind usa 64 MB de buffer do RetroArch — pode regredir até ~30 segundos dependendo do jogo.

💜 Triggers de likes

O sistema acumula likes por usuário e dispara ações em milestones configuráveis. Útil para recompensar fãs recorrentes.

Configuração

Em ✏️ Editar → aba Likes, defina o threshold e a sequência para cada trigger:

CampoDescrição
LikesThreshold acumulado por usuário (ex: 50, 100, 500)
SequênciaMesma sintaxe de gifts (ex: B:150)
Label / TTSTexto exibido e narrado ao atingir o milestone
ℹ️
O contador é por usuário e por sessão — reinicia quando a página é recarregada.

💬 Palavras-chave no chat

Detecta frases específicas no chat ao vivo e dispara sequências. Tem cooldown de 10 segundos por palavra para evitar spam.

Configuração

Em ✏️ Editar → aba Palavras-chave:

CampoDescrição
Palavra/fraseTexto que será buscado no chat (case insensitive)
SequênciaAção disparada quando a frase for detectada
Label / TTSFeedback visual e de voz

Exemplos de uso: votações (vai direita / vai esquerda), ativar cheats por comando da audiência, easter eggs escondidos.

⭐ Power-ups (cheats via RAM)

Escreve diretamente na Work RAM do SNES para aplicar cheats instantâneos — sem pausar o jogo.

Formato

ENDEREÇO:VALOR

Ambos em hexadecimal. Exemplo: 7E00CE:02 (Fire Flower no SMB All-Stars).

Como encontrar endereços

Endereços SNES no formato 0x7Exxxx → a posição na WRAM é addr & 0x1FFFF. Consulte GameFAQs ou o SNES RAM Map do seu jogo para os endereços corretos por ROM e região.

Método de escrita

Método direto (preferido): _retro_get_memory_data(2) → ponteiro WRAM → Module.HEAPU8[ptr + offset] = valor. Instantâneo.

Fallback automático: se o método direto não estiver disponível, salva o estado, faz patch binário no bloco RAM e recarrega.

⚠️
Valores variam por ROM e região Um endereço válido para a versão americana pode ser diferente na versão japonesa ou europeia. Sempre teste antes da live.

Criar preset personalizado

Abra ✏️ Editar, faça as alterações desejadas e clique Salvar. Um preset custom é criado automaticamente (os presets embutidos não são sobrescritos). Para nomear, edite diretamente em app.js adicionando uma entrada em PRESETS.

🔊 Narração TTS

O app usa a Web Speech API nativa do browser para narrar eventos em português brasileiro.

Ativação

Por política dos browsers, o TTS só funciona após a primeira interação do usuário com a página (clique, tecla ou toque). Assim que você interagir, a voz será ativada automaticamente.

Seleção de voz

O app seleciona automaticamente uma voz localService em pt-BR. Vozes locais garantem funcionamento offline e evitam falhas silenciosas das vozes online do Google no Chrome macOS.

⚠️
Chrome macOS As vozes online do Google (pt-BR) falham silenciosamente nesse ambiente. O app detecta isso e usa apenas vozes locais. Se nenhuma voz pt-BR local estiver disponível, o TTS será desativado automaticamente.

Personalizar o texto narrado

Cada gift tem um campo TTS no editor. Deixe vazio para silenciar aquele gift. Use texto natural — o narrador usa a voz padrão pt-BR do sistema.

🔧 Requisitos técnicos

Browser suportados

Safari⚠️ LimitadoSharedArrayBuffer restrito

SharedArrayBuffer

O Nostalgist.js usa Emscripten pthreads (threads WASM), que exige SharedArrayBuffer. Esse recurso só está disponível em contextos seguros com os headers:

Cross-Origin-Opener-Policy: same-origin
Cross-Origin-Embedder-Policy: credentialless

Rodando local — use python3 server.py. O servidor já adiciona esses headers.

GitHub Pages — sem controle de headers. O coi-serviceworker.js intercepta as respostas via Service Worker e injeta os headers automaticamente.

Não funciona via file:// — sempre use um servidor HTTP.

ℹ️
Se o emulador não carregar, abra o DevTools (F12) → Console. A mensagem "SharedArrayBuffer is not defined" indica que os headers não estão presentes.

🌐 Hospedar no GitHub Pages

O projeto é estático — basta fazer upload dos arquivos para um repositório público.

1
Crie um repositório público no GitHub Ex: webtiktoklive
2
Faça o commit dos arquivos index.html, app.html, ajuda.html, style.css, app.js, coi-serviceworker.js, lib/, assets/
3
Ative o GitHub Pages Settings → Pages → Branch: main → / (root) → Save
4
Aguarde ~1 minuto O site estará disponível em https://seuusuario.github.io/webtiktoklive/
